    2. 商户接口

    1. 商户接口

      1. 商户进件接口

        1. 地址

    http://internal.congmingpay.com/internal/registernewmerchant.do

    1. 请求数据
    字段名 变量名 必填 类型 示例 描述
    订单编号 order_id 是 String（32） abc20181116142365897 申请商户进件单号
    商户全称 shop_name 是 String（32） 北京市西城区张三便利店 商户全称，营业执照名称
    商户简称 shop_nickname 是 String（32） 张三小店 商户简称，不能少于4个字，特殊需人工处理
    负责人 shop_keeper 是 String（20） 张三 负责人姓名
    负责人电话 keeper_phone 是 String（20） 1888888888 负责人电话，商户管理系统登陆账号
    客服电话 shop_phone 否 String（20） 01087945612 客服电话
    邮箱 email 否 String（32） abc\@163.com 邮箱
    省 province 是 String（20） 北京市 店铺所在省
    市 city 是 String（20） 北京市 店铺所在市
    区/县 area 是 String（20） 西城区 店铺所在区
    详细地址 shop_address 是 String（128） 建国路东条胡同101号 店铺详细地址
    渠道 pay_type 是 String（20） suixingfu 需开通渠道，包括： sub：标准微信支付宝 lc：富有三方通道 wft_xy：兴业银行通道 xtban：邢台银行通道 yirongma：和融通三方通道 suixingfu：随行付三方通道
    商户类型 business_type 是 String（2） 1 商户类型，1：企业商户；2个体工商户；3：个人商户；
    营业执照号 licence_no 选择 String（32） 91320000MA1M915G3J 企业或个体户营业执照号
    起始时间 licence_begin_date 选择 String（20） 2005-12-30 营业执照起始时间，格式yyyy-MM-dd
    截止时间 licence_expire_date 选择 String（20） 2037-12-29 营业执照截止时间，格式yyyy-MM-dd
    法人姓名 artif_name 选择 String（20） 张三
    法人手机号 artif_phone 选择 String（20） 1888888888 法人手机号，企业必填
    法人身份证号 artif_identity 选择 String（32） 134123456789452136 法人身份证号，企业必填
    微信费率 rate_wx 是 String（20） 3.8 微信费率，单位千分之一
    支付宝费率 rate_alipay 是 String（20） 3.8 支付宝费率，单位千分之一
    结算人姓名 card_name 是 String（20） 张三 结算人姓名
    结算卡号 card 是 String（32） 6228080466514512 结算银行卡号
    结算人手机号 card_phone 是 String（20） 1888888888 结算人手机号
    结算银行 bank_name 是 String（32） 中国农业银行 结算银行
    结算开户行 bank_address 是 String（64） 中国农业银行股份有限公司西直门分行 结算卡开户行
    结算联行号 bank_add_no 是 String（20） 308305008197 结算卡开户行联行号
    结算人身份证号 identity 是 String（32） 134123456789452136 结算人身份证号
    经营范围 type 是 String（20） 美食 商户一级经营范围，参考附件经营范围表
    经营范围 classify 是 String（20） 家常菜 商户二级经营范围，参考附件经营范围表
    备注信息 description 否 String（128） 备注信息 备注信息，如配置appid等
    回调地址 notify_url 否 String（256） http://www.congmingpay.com 商户审核通过回调地址
    合作商号 agent_id 是 String（32） hb_bd_0001 合作商商户号
    签名 sign 是 String（32） EABDKFSKDFJKLSDLJFKSDFJFSKLD 签名，签名方式与微信相同，参考5.1
    1. 响应数据
    字段名 变量名 必填 类型 示例 描述
    进件状态 result_code 是 String（20） success 进件状态：success：进件成功；fail：进件失败
    失败说明 error_msg 否 String（32） 负责人电话必填 失败时的原因说明
    商户号 shop_id 选择 String（32） c2e17899960c83c2aeca03c37b13ade0 进件成功时，返回的商户号。

    1. 商户开通成功异步回调接口

      1. 说明

    异步通知需在进件接口中notify_url字段有值，且可以外网正常访问，服务器在商户正常审核通过后，会回调此地址，并采用post方式发送json格式字符串

    1. 请求数据
    字段名 变量名 必填 类型 示例 描述
    进件状态 result_code 是 String（20） success 进件状态：success：进件成功；fail：进件失败
    失败说明 error_msg 否 String（32） 负责人电话必填 失败时的原因说明
    订单号 order_id 选择 String（32） abc20181116142365897 进件时提交的订单号
    商户号 shop_id 选择 String（32） c2e17899960c83c2aeca03c37b13ade0 进件成功时，返回的商户号。
    1. 响应数据
    字段名 变量名 必填 类型 示例 描述
    进件状态 result_code 是 String（20） success 进件状态：success：进件成功；fail：进件失败
